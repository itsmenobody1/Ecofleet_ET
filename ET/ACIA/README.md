# Autonomous Cost Intelligence Agent (ACIA)

> **ACIA doesn't show dashboards. It finds ₹ leaks and takes corrective actions through approval workflows.**

## 🚀 Quick Start

```bash
# Install dependencies
pip install -r requirements.txt

# Run the app
python -m streamlit run app.py
```

## 🧩 Architecture

| Layer | Files | Purpose |
|-------|-------|---------|
| **Config** | `config/settings.py`, `config/prompts.py` | API keys, thresholds, LLM prompts |
| **Core** | `core/data_loader.py`, `core/cost_calculator.py`, `core/anomaly_engine.py`, `core/action_engine.py` | Data ingestion, ₹ math, anomaly detection, action execution |
| **Agents** | `agents/spend_agent.py`, `agents/sla_agent.py`, `agents/resource_agent.py`, `agents/finance_agent.py` | Specialised cost intelligence agents |
| **LLM** | `llm/llm_client.py`, `llm/reasoning.py` | OpenAI integration + rule-based fallback |
| **Workflows** | `workflows/approval_manager.py`, `workflows/action_logs.py` | Approval state, JSONL action log |
| **UI** | `ui/*.py` | Streamlit pages |

## 🤖 Agents

| Agent | Data Sources | Detects |
|-------|-------------|---------|
| Spend Agent | procurement.csv, saas_usage.csv | Price overruns, unused SaaS licences |
| SLA Agent | tickets.csv | SLA breach risk & penalty exposure |
| Resource Agent | cloud_usage.csv | Idle cloud servers |
| Finance Agent | invoices.csv | Duplicate payments |

## 🔑 Optional LLM Setup

Create a `.env` file:

```
OPENAI_API_KEY=sk-...
LLM_MODEL=gpt-4o-mini
```

Without an API key the app runs fully in rule-based mode — all findings, cost math, and playbooks are generated by the anomaly engine.

## 📁 Sample Data

Five pre-built CSVs in `data/sample_data/` contain realistic enterprise anomalies:
- **procurement.csv** — vendor price overruns across 15 purchase orders
- **cloud_usage.csv** — 7 idle servers with full cost breakdown
- **saas_usage.csv** — 6 tools with >20% unused seats
- **tickets.csv** — 12 tickets close to or past SLA
- **invoices.csv** — 5 duplicate invoice pairs
